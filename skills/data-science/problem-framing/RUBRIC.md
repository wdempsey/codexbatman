# Rubric: Problem Framing

## Passing Standard

The output passes if it:

- states the objective type clearly
- defines the decision context and success metrics
- documents constraints and risks explicitly
- names concrete data requirements
- provides a usable artifact for audit approval

## Failure Modes

Fail if the output:

- leaves the objective type ambiguous
- uses vague metrics
- omits risk documentation
- jumps to solution design or modeling
