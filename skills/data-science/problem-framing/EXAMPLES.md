# Examples: Problem Framing

## Example 1

User request:

> Frame this churn prediction project before we touch the data.

Expected behavior:

- classify the objective type
- define the decision context
- produce a formal framing artifact

## Example 2

User request:

> I think this is a causal policy question, but I am not sure. Help me formalize it.

Expected behavior:

- distinguish causal from predictive framing
- surface identification assumptions
- document risks before any audit work

## Example 3

User request:

> Turn this loose project description into a problem-framing artifact I can review with my team.

Expected behavior:

- structure the request into the canonical sections
- expose ambiguity rather than filling gaps silently
