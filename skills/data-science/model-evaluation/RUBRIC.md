# Rubric: Model Evaluation

## What Good Execution Looks Like

- Performance is interpreted against decision requirements.
- Error analysis identifies concrete failure patterns.
- Robustness checks test key assumptions.
- Recommendation is evidence-based and actionable.

## Minimum Acceptable Outputs

- evaluation report
- metric summary
- error analysis
- robustness check summary
- recommendation/next-step note

## Common Mistakes

- one-metric evaluation
- no subgroup checks
- no robustness tests
- recommendation without rationale
- no action plan

## Review Checklist

- Are metrics interpreted in context of framing criteria?
- Is error analysis specific and decision-relevant?
- Are subgroup disparities evaluated and reported?
- Are robustness checks sufficient for the use case?
- Is the recommendation explicit with next steps?
