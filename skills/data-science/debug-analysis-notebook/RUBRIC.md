# Rubric: Debug Analysis Notebook

## What Good Execution Looks Like

- Debug is evidence-led and minimally invasive.
- Root cause is isolated before changes are applied.
- Leakage and split integrity are rechecked.
- Verification steps are explicit and reproducible.

## Minimum Acceptable Outputs

- symptom summary
- reproduction steps
- root-cause analysis
- minimal fix plan
- verification checklist

## Common Mistakes

- full notebook rewrite without diagnosis
- no reproduction path
- no contamination checks
- no verification plan
- fixing outputs without fixing cause

## Review Checklist

- Is the failure clearly reproduced?
- Is root cause linked to evidence?
- Are fixes minimal and targeted?
- Were leakage/split risks reassessed?
- Is there a clear post-fix verification plan?
