# Rubric: Gradient Boosting

## What Successful Understanding Looks Like

- The student can explain gradient boosting as sequential correction of errors.
- The student can describe why it often performs well on tabular data.
- The student can name tuning sensitivity and overfitting as practical risks.
- The student understands why it usually comes after a baseline and often after random forest in a beginner workflow.

## Minimum Acceptable Understanding

- can explain when gradient boosting is appropriate
- can name one reason it is more demanding than a baseline
- can describe one tuning-related caveat

## Common Misconceptions

- boosting is just random forest with more trees
- stronger predictive performance means it should always be the first method tried
- more boosting rounds always improve the model
- feature importance from boosting proves causal influence

## Readiness To Return To Workflow

- can explain when gradient boosting is appropriate
- can name one tradeoff versus a simpler baseline or random forest
- can describe one failure mode or caveat, such as overfitting from poor tuning
