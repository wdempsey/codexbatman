# Skill: Stakeholder Update

## Purpose

Turn project state into a clear external update.

This skill is for status communication, not full project synthesis. It converts existing project memory into a message tailored to the audience.

## When to Use

Use this skill when:

- a manager needs a weekly or monthly stakeholder update
- a funder or leadership summary is due
- a collaborator needs a concise project status note
- a project dashboard must be translated into audience-appropriate prose

## Required Inputs

- Intended audience
- Update purpose
- Time window or recent milestone
- Current project status artifacts

Prefer these sources in order:

- current dashboard
- latest weekly review
- current research design or project record
- recent meeting notes if needed

## Procedure

### Step 1: Define the Audience

Determine:

- who will read the update
- what they care about
- what level of detail is appropriate
- what content is sensitive or unnecessary

### Step 2: Gather Authoritative Context

Read the strongest available project sources first.

Do not build an update from memory if current project artifacts exist.

### Step 3: Select the Core Message

Identify:

- progress since the last update
- current status
- major risks or dependencies
- upcoming milestones
- any asks or decisions needed

### Step 4: Tailor the Framing

Adjust tone and emphasis by audience:

- collaborators need operational clarity
- leaders need status, risk, and decisions
- funders need outcomes, milestones, and credible next steps

### Step 5: Draft a Tight Update

Keep the update concise and factual. Prefer explicit uncertainty over inflated confidence.

## Output Format

Produce:

### 1. Audience and Purpose

### 2. Key Points to Communicate

### 3. Draft Update

### 4. Risks, Dependencies, or Asks

### 5. Confidence Notes

## Guardrails

- Do not introduce facts not supported by current project artifacts.
- Do not expose internal personnel commentary or sensitive operational detail to broad stakeholders.
- Do not write generic progress language when concrete milestones are available.
- Do not bury the actual ask if a decision is needed.

## Escalation Conditions

Stop and ask for direction if:

- the target audience is unclear
- the latest project state is stale or contradictory
- sensitive content may require human review
- the user needs a full proposal or report instead of a status update
