# Examples: Weekly Review

## Example 1

User request:

> Generate this week’s project review and update the dashboard draft.

Expected behavior:

- identify the review window
- gather current project sources
- update the dashboard from the prior baseline
- surface actions, blockers, and confidence notes

## Example 2

User request:

> I have transcripts and email, but WhatsApp is unavailable. Do the weekly review anyway.

Expected behavior:

- proceed with graceful degradation
- note missing source coverage explicitly
- avoid implying full coverage

## Example 3

User request:

> Give me a manager-ready review for the last two weeks, focused on strategic priorities and deadlines.

Expected behavior:

- adjust the review window
- organize around objectives and deadlines
- keep the output decision-oriented
