# Rubric: Inbox Triage

## Passing Standard

The output passes if it:

- scopes triage to the right project and time window
- groups related emails by thread
- extracts owners, actions, waiting-fors, and deadlines
- separates signal from low-value inbox noise
- produces outputs usable for project tracking or meeting prep

## Failure Modes

Fail if the output:

- lists emails without synthesizing them
- misses thread-level decisions
- mixes FYI items with action items
- omits owners or timing
- over-classifies irrelevant mail as project-critical
