# Rubric: Project Setup

## Passing Standard

The output passes if it:

- accurately reflects the current project state
- proposes an additive, minimal structure
- identifies the core artifacts needed for downstream workflows
- maps real external systems instead of hypothetical ones
- flags risks and missing inputs clearly

## Failure Modes

Fail if the output:

- assumes a structure without inspecting the project
- proposes reorganization without user approval
- omits source-of-truth documents or update locations
- ignores sensitive-data constraints
- marks the project ready when key dependencies are missing
