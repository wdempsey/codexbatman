# How Codex Modes Work

When you first launch Codex, you are in **Default mode**, which is usually the safest starting point.

## The Three Modes

| Mode | What Codex Does | What You See |
|------|-----------------|-------------|
| Default | Proposes edits and waits for confirmation | Normal prompt |
| Plan Mode | Explores and proposes without changing files | `(plan)` label |
| Auto-Accept | Applies edits automatically | accept-edits indicator |

## How to Switch

Use the controls available in your Codex environment to move between modes.

The core idea is:

- use **Default** when you want review before edits
- use **Plan Mode** when you want Codex to think before acting
- use **Auto-Accept** only when the task is routine and you trust the change path

## When Students Usually Use Each Mode

| Situation | Best Mode |
|-----------|-----------|
| "I want help understanding the workflow first" | Plan Mode |
| "I want guided help and want to review edits" | Default |
| "I trust this small cleanup task" | Auto-Accept |
