# Workflow for Managing Data Science

A project-centered workflow for managers leading data science, analytics, and research work. Connect email, meetings, docs, Slack, Dropbox, and private notes into one system for project awareness, follow-up tracking, and weekly review.

!!! note "You should read this after setup"
    Before using this workflow, you should already have:

    - Codex installed and working
    - a basic context file or project instructions file
    - access to your core systems (Google, Slack, Dropbox, notes)

    If not, start with the setup guides first, then return here.

## 1. Why this workflow exists

- The manager problem: too many parallel projects with fragmented context.
- Data science work creates fragmentation across email, meetings, docs, Slack, files, and personal notes.
- Inbox-first systems are not enough because they optimize message handling, not project-state awareness.

## 2. Core idea

- Projects are primary.
- Communication and notes feed project memory.
- The assistant helps operate the workflow, but is not the workflow itself.

## 3. What this workflow helps you do

- Daily prioritization.
- Inbox triage.
- Meeting prep and follow-up.
- Project tracking.
- Communication support.
- Weekly review.

## 4. The components

- Project registry.
- Gmail layer.
- Calendar layer.
- Drive / Docs layer.
- Slack layer.
- Dropbox layer.
- Personal notes layer (Obsidian).
- Task / waiting-for layer.
- Review layer.

## 5. How to adopt it

- Phase 0: Prerequisites.
- Phase 1: Core Google setup.
- Phase 2: Project-centered organization.
- Phase 3: Slack integration.
- Phase 4: Dropbox integration.
- Phase 5: Obsidian integration.
- Phase 6: Full review workflow.

## 6. Example use cases

- Daily brief.
- Project update from email.
- Meeting prep using shared and private context.
- Weekly review.
- Follow-up and waiting-for tracking.

## 7. Design rules

- Human-supervised by default.
- Shared memory vs private memory.
- Projects remain central.
- Build in phases.

## 8. Where this fits in the broader system

- Relationship to Project Management: this is a manager-facing operating workflow that applies project-management logic to daily execution.
- Relationship to Executive Assistant: this extends inbox and calendar support into project-centered managerial synthesis.
- Relationship to setup guides and Toolkit pages: setup/toolkit pages explain installation and components, while this page explains operating behavior.
